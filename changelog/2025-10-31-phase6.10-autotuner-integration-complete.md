## Part 1: Device-Aware GEMM Factory

**Goal**: Tensors should automatically create the correct kernel based on their device placement.

### Implementation: `CudaGemmFactory.{h,cu}`

**Architecture**:
```cpp
// In IQ4_NLTensor.cpp
std::unique_ptr<ITensorGemm> IQ4_NLTensor::createGemm()
{
    if (device_idx_ >= 0) {  // GPU device
        auto& devices = DeviceManager::instance().devices();
        switch (devices[device_idx_].type) {
            case GPU_CUDA:
                return llaminar::v2::kernels::cuda::createCudaGemm(this);
            case GPU_ROCM:
                // Future: ROCm support
                break;
        }
    } else {  // CPU device
        return llaminar::v2::kernels::createAutoTunedGemm(this);
    }
}
```

**CudaGemmFactory Design**:
- **Forward declarations only** in `.h` (avoids MPI header conflicts)
- **Local ITensorGemm definition** in `.cu` (separate compilation unit)
- **CudaGemmKernel wrapper class** implements ITensorGemm interface
- **Integrates with auto-tuner** for optimal tile selection

**Files Created**:
- `src/v2/kernels/cuda/CudaGemmFactory.h` (~95 lines)
- `src/v2/kernels/cuda/CudaGemmFactory.cu` (~150 lines)

**Files Modified**:
- `src/v2/tensors/IQ4_NLTensor.cpp` (added device-aware routing)
- `src/v2/CMakeLists.txt` (added CudaGemmFactory.cu to CUDA sources)

**Build Status**: ✅ Compiles cleanly (cuda_backend + llaminar2_core)

### Key Design Decisions

**Problem**: CUDA files cannot include MPI headers (NVCC doesn't support MPI.h)

**Solution**: Forward declarations + local interface definition
```cpp
// CudaGemmFactory.h (NO MPI headers)
namespace llaminar2 { class IQ4_NLTensor; }
namespace llaminar2::kernels::cuda {
    std::unique_ptr<void> createCudaGemm(const IQ4_NLTensor* tensor);
}

// CudaGemmFactory.cu (defines ITensorGemm locally)
namespace llaminar2 {
    class ITensorGemm { /* local definition */ };
    
    class CudaGemmKernel : public ITensorGemm {
        bool multiply(...) override;
    };
}
```

**Result**: Zero header pollution, clean separation of concerns.

## Part 2: CUDABackend Integration

**Goal**: Replace legacy `launchIQ4NLGemm` with auto-tuned variant launcher.

### Before (Legacy System)

```cpp
// src/v2/backends/cuda/CUDABackend.cu
#include "kernels/cuda/IQ4_NL_Gemm.h"

bool CUDABackend::gemmIQ4NL(
    void* A_device, void* B_device, void* C_device,
    int m, int n, int k, int device_id)
{
    // Cast pointers
    const float* A = static_cast<const float*>(A_device);
    const cuda::IQ4_NLBlock* B = static_cast<const cuda::IQ4_NLBlock*>(B_device);
    float* C = static_cast<float*>(C_device);
    
    // Fixed-tile legacy kernel (16×16 only)
    cudaError_t err = cuda::launchIQ4NLGemm(A, B, C, m, n, k, 0);
    
    if (err != cudaSuccess) { /* error handling */ }
    return true;
}
```

**Limitations**:
- ❌ Fixed 16×16 tile (suboptimal for many shapes)
- ❌ No auto-tuning
- ❌ Legacy API

### After (Auto-Tuned Variant System)

```cpp
// src/v2/backends/cuda/CUDABackend.cu
#include "kernels/cuda/IQ4_NL_Gemm.h"
#include "kernels/cuda/IQ4_NL_BlockDecoder.h"
#include "kernels/cuda/CudaGemmVariants.h"
#include "kernels/cuda/CudaGemmAutoTuner.h"

bool CUDABackend::gemmIQ4NL(
    void* A_device, void* B_device, void* C_device,
    int m, int n, int k, int device_id)
{
    // Cast pointers
    const float* A = static_cast<const float*>(A_device);
    const cuda::IQ4_NLBlock* B = static_cast<const cuda::IQ4_NLBlock*>(B_device);
    float* C = static_cast<float*>(C_device);
    
    // Get optimal tile configuration from auto-tuner
    auto &autotuner = cuda::CudaGemmAutoTuner::instance();
    auto config = autotuner.getOptimalConfig(m, n, k);
    
    // Create decoder instance (holds B pointer and dimensions)
    cuda::IQ4_NL_Decoder<cuda::IQ4_NLBlock> decoder(B, n, k);
    
    // Launch optimized variant kernel
    cudaError_t err = cuda::launchIQ4NLGemmVariant(
        A, C, m, n, k, config, decoder, 0);
    
    if (err != cudaSuccess) { /* error handling */ }
    return true;
}
```

**Improvements**:
- ✅ Auto-tuned tile sizes (16×16, 32×32, 64×64, 128×32, etc.)
- ✅ Shape-aware selection (small/medium/large matrices)
- ✅ Generic decoder pattern (ready for Q6_K, Q8_0)
- ✅ Zero performance regression (same decoder logic)

## Integration Points

### 1. IQ4_NLTensor Integration

**Before**: Always created CPU kernel
```cpp
std::unique_ptr<ITensorGemm> IQ4_NLTensor::createGemm() {
```

**After**: Device-aware routing
```cpp
std::unique_ptr<ITensorGemm> IQ4_NLTensor::createGemm() {
    if (device_idx_ >= 0) {  // GPU
        switch (DeviceManager::instance().devices()[device_idx_].type) {
            case GPU_CUDA: return createCudaGemm(this);
            case GPU_ROCM: // Future
        }
    } else {  // CPU
        return createAutoTunedGemm(this);
    }
}
```

### 2. CUDABackend Integration

**API**: `bool gemmIQ4NL(void* A, void* B, void* C, int m, int n, int k, int device_id)`

**Flow**:
1. Cast pointers to typed device memory
2. Query auto-tuner for optimal tile config
3. Create IQ4_NL_Decoder instance
4. Launch variant kernel with selected config
5. Synchronize and check for errors

**Error Handling**:
- Invalid dimensions (k not multiple of 32)
- CUDA memory errors
- Kernel launch failures
- Execution errors

### 3. Auto-Tuner Integration

**Singleton Pattern**: `CudaGemmAutoTuner::instance()`

**API**: `CudaGemmConfig getOptimalConfig(int m, int n, int k)`

**Selection Process**:
1. **Shape classification**: Small/medium/large based on m×n×k
2. **Hierarchical filtering**: Eliminate invalid configs
3. **Analytical ranking**: Compute score based on:
   - Compute density (tile_m × tile_n × tile_k)
   - Thread utilization (threads_x × threads_y)
   - Memory access pattern (K_BLOCK_SIZE)
4. **Return best config** (highest score)

**Fallback**: If no valid config, returns safe default (16×16×32)

## Lessons Learned

### 1. MPI Header Conflicts in CUDA

**Problem**: CUDA .cu files cannot include MPI headers (NVCC incompatibility)

**Solution**: Forward declarations + local interface definitions
- Declare types in .h (no includes)
- Define interfaces locally in .cu
- Use separate compilation units

**Lesson**: Keep CUDA code isolated from complex C++ libraries.

### 2. Legacy Code Accumulation

**Problem**: 194-line IQ4_NL_Gemm.cu was 79% obsolete code

**Solution**: Regular refactoring and cleanup
- Identify essential vs obsolete code
- Migrate production code first
- Remove legacy code second
- Document changes thoroughly

**Lesson**: Clean up immediately after migration, not "later".

### 3. Generic Patterns Pay Off

**Problem**: Adding Q6_K, Q8_0 would require duplicating 200+ kernels

**Solution**: Generic decoder pattern (IBlockDecoder interface)
- Single kernel template
- Format-specific decoders
- Zero overhead (inline methods)

**Lesson**: Invest in generic infrastructure early, not after N formats.

### 4. Auto-Tuner Value Increases With Scale

**Small matrices**: Auto-tuner overhead ~= performance gain
**Large matrices**: Auto-tuner overhead << performance gain

**Lesson**: Auto-tuning most valuable for large-scale inference workloads.
