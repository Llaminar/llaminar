## Implementation Details

### Core Changes

**1. Device-Aware Helper Method** (`src/v2/pipelines/qwen/Qwen2Pipeline.{h,cpp}`)
- Added `prepareActivationForDevice()` method (~70 lines)
- Handles cross-device activation transfers transparently
- Uses residual buffer as transfer staging area (safe - not in use during transfer)
- Validates buffer size before transfer
- Includes debug logging for transfer tracking

**Key logic**:
```cpp
TensorBase* Qwen2Pipeline::prepareActivationForDevice(
    TensorBase* activation, int target_device, const std::string& context)
{
    // 1. Null check
    if (!activation) return nullptr;
    
    // 2. Skip transfer if already on target device
    int current_device = activation->device_index();
    if (current_device == target_device) return activation;
    
    // 3. Get target device buffer pool (Phase 4.1)
    auto& target_buffers = getBuffersForDevice(target_device);
    auto* transfer_buffer = target_buffers.residual.get();
    
    // 4. Validate buffer size
    size_t act_size = activation->size_bytes();
    size_t buf_size = transfer_buffer->size_bytes();
    if (act_size > buf_size) return nullptr;
    
    // 5. Perform transfer (Phase 4.2 copy)
    std::cout << "Transferring " << context << ": device "
              << current_device << " → " << target_device << "\n";
    if (!transfer_buffer->copyFrom(activation)) return nullptr;
    
    return transfer_buffer;
}
```

**2. Attention Block Updates** (`attention_block()`)
- Query attention weight device: `int attn_device = getWeightDevice("attn_q", -1);`
- Transfer activation if needed: `prepareActivationForDevice(current_hidden_.get(), attn_device, "attention_input")`
- Use device-specific buffer pool: `auto& buffers = getBuffersForDevice(attn_device);`
- Execute all operations on `attn_device` (not hardcoded `device_idx_`)
- Update `current_hidden_` device index after completion: `current_hidden_->set_device(attn_device);`

**3. FFN Block Updates** (`ffn_block()`)
- Query FFN weight device: `int ffn_device = getWeightDevice("ffn_gate", -1);`
- Transfer activation if needed (may differ from attention device!)
- Use device-specific buffer pool
- Execute on FFN device
- Update activation device index

**4. Temporary Tensor Creation**
- Created temporary `normalized_hidden` tensor for in-place RMSNorm
- Allocated on target device for optimal locality
- Prevents mutation of input activation (cleaner separation)

### Architecture Pattern

**Device-aware execution flow**:
```
1. Query weight device (Phase 4.1): getWeightDevice("weight_name", layer_idx)
2. Transfer activation if needed (Phase 4.3): prepareActivationForDevice(...)
3. Get device-specific buffers (Phase 4.1): getBuffersForDevice(device)
4. Execute kernels on target device (Phase 4.2): kernel->multiply(..., device_idx)
5. Update activation device state: current_hidden_->set_device(device)
```

**Result stays on weight device** - caller can transfer back if needed for next operation.

## Design Decisions

**1. Execute on Weight Device** (not hardcoded `device_idx_`)
- Operations execute where weights are located
- Reduces unnecessary weight transfers (weights >> activations)
- Enables per-layer heterogeneous execution

**2. Result Stays on Weight Device**
- Caller can transfer back if next operation needs different device
- Avoids unnecessary round-trip transfers
- Clear ownership semantics

**3. Use Residual Buffer as Transfer Staging**
- Safe: Not in use during attention/FFN computation
- Efficient: Reuses pre-allocated buffer
- Sized for worst-case (max_seq_len × d_model)

**4. Temporary Normalized Hidden Tensor**
- Prevents mutation of input activation
- Cleaner separation of concerns
- Allocated on target device for locality

## Integration with Phase 4.1 and 4.2
